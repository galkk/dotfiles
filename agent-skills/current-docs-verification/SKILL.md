# Current Docs Verification

- Verify from current official docs before asserting support, limitations, latest behavior, model availability, pricing-sensitive details, or recently changed product behavior.
- Prefer primary sources: vendor docs, official changelogs, official repositories, standards, or release notes.
- For OpenAI products, use the `openai-docs` skill first. Fall back only to official OpenAI domains if needed.
- For Anthropic, Google, GitHub, cloud providers, CLIs, and libraries, use official documentation or official repositories before third-party summaries.
- Cite direct links for externally verified claims. If no official source confirms the claim, say that it is unverified.
- Keep verification scoped to the claim being answered; do not turn a simple capability check into broad product research unless requested.
